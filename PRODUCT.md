# Neuron Product Context

Neuron is an open-source coding-agent CLI for cloud and local model providers. It should feel capable, calm, and provider-neutral: the identity is Neuron, while model/provider names only appear when they describe real configuration or billing choices.

## Users

- Developers who want an agentic terminal workflow without being locked into a single model provider.
- Power users who care about local models, API routing, provider profiles, memory, skills, permissions, and cost visibility.
- New users who need a guided first run that feels premium, quick, and understandable.

## Product Purpose

Neuron helps users understand a codebase, make edits with permission, run commands, manage skills and memory, and route work across providers. The UI should make powerful workflows feel safe, inspectable, and enjoyable.

## Voice

Clear, warm, direct, and lightly playful. Avoid vendor identity leakage in product copy. Use "Neuron" when the app or agent is acting; use provider names only for provider-specific setup, auth, billing, model IDs, or integrations.
