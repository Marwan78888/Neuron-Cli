# Neuron Design Context

## Experience Principles

- Premium terminal product: dense enough for power users, polished enough for first-run confidence.
- Safety without fear: permission prompts should explain what is happening and what the user can do next.
- Provider-neutral identity: Neuron is the product voice; Anthropic, OpenAI, Gemini, Ollama, and others appear only as functional choices.
- Reduce noise: keep onboarding and prompts concise, with progress cues and concrete actions.

## Interface Notes

- Prefer compact panes, clear step labels, and short helper text.
- Avoid marketing copy inside operational UI.
- Keep status and permission language action-oriented: "Neuron requested...", "Tell Neuron...", "Review before running generated code."
- Use tasteful moments of delight in onboarding, but keep the flow fast.

## UI Polish Priorities

- Onboarding should communicate progress, personalization, auth/provider choice, security, and terminal ergonomics without feeling like a chore.
- Skills and memory surfaces should emphasize usefulness without spending unnecessary context.
- Error text should be specific, calm, and recoverable.
